1. مقدمة — لماذا AI وكيف يُستخدم في PMS

نظام PMS يدمج Google Gemini كخدمة خارجية (External API) — أي أن التعلم الآلي لا يعمل داخل السيرفر المحلي. السيرفر يرسل نصاً (prompt) إلى Gemini ويستقبل JSON منظّماً، ثم يتحقق منه ويخزّنه في قاعدة البيانات. هذا يجعل الذكاء الاصطناعي مساعداً للطالب وليس نظاماً يتحكم تلقائياً بالمشروع.

مبدأ التصميم: Human-in-the-loop — الطالب يقرأ الاقتراحات، يختار، يحفظ، أو ينشئ مشروعاً. المشرف والفريق يرون نشاط التوليد في سجل المشروع والإشعارات. لا يُنشأ مشروع أو مهمة دون إجراء صريح من المستخدم.

ميزتان رئيسيتان:

الميزةالهدفالمستخدمالمخرجات
Project Ideation اقتراح 3 أفكار مشاريع تخرج من اهتمامات الطالب طالب فقط (role:student) 3 بطاقات: اسم، هدف، تقنيات + إمكانية حفظ (bookmark)
AI Task Generation تقسيم وصف المشروع إلى مهام قابلة للتنفيذ طالب عضو مقبول في المشروع 10 مهام (قابلة للتعديل) مع estimated_hours و ai_generated=true

2. البنية المعمارية — طبقات النظام

كل طلب AI يمر بنفس النمط المعماري للنظام:

React Page (ProjectIdeation.jsx / TasksTab.jsx) → apiFetch + Bearer Token (Sanctum) → Laravel Route + Middleware (auth, role, throttle) → Controller (HTTP validation) → Service (AIIdeationService / AITaskService) → Http::post → Google Gemini REST API → parse JSON + Laravel Validator → DB Transaction (tasks / bookmarks / logs) → JSON Response → React state update

4. تكامل Gemini API — الطلب والاستجابة

كلا الخدمتين (AIIdeationService و AITaskService) تستخدمان نفس نمط HTTP:

4.1 رؤوس الطلب (Headers)

Content-Type: application/json
x-goog-api-key: {GEMINI_API_KEY}

4.2 جسم الطلب (Body)

{
  "contents": [
    { "parts": [ { "text": "{PROMPT_BUILT_FROM_TEMPLATE}" } ] }
  ],
  "generationConfig": {
    "temperature": 0.7,
    "maxOutputTokens": 2048 | 4096,
    "responseMimeType": "application/json"
  }
}
  • Ideation: maxOutputTokens: 2048 — كافٍ لـ 3 اقتراحات JSON.
  • Tasks: maxOutputTokens: 4096 — لـ 10 مهام مع أوصاف.
  • responseMimeType: يفرض على Gemini إرجاع JSON خام (بدون markdown).
  • temperature 0.7: توازن بين الإبداع والثبات في الاقتراحات.

4.3 استخراج النص من الاستجابة

response.json() → candidates[0].content.parts[0].text → strip optional ```json fences → json_decode → Laravel Validator (schema rules) → return structured array to Controller

4.4 منطق إعادة المحاولة (Retry)

يُعاد الطلب تلقائياً (حتى 3 مرات) فقط عند:

  • ConnectionException — انقطاع شبكة.
  • RequestException مع HTTP status ≥ 500 أو بدون status.

لا يُعاد عند 401/403/404/422 — هذه أخطاء تكوين أو طلب غير صالح.

4.5 رموز HTTP المُرجَعة للواجهة

حالة Gemini / ServiceHTTP للعميلرسالة للمستخدم
نجاح200 / 201success message + data
Validation (interests قصيرة، description < 20، regenerate ممنوع)422رسالة validation محددة
5xx أو ConnectionException503«AI service temporarily unavailable…»
401/403/404 من Gemini500configuration / model unavailable
JSON غير صالح من AI500«Unable to process AI response…»

5. ميزة 1: اقتراح أفكار المشاريع (Project Ideation)

5.1 مسار المستخدم (UX)

  1. الطالب يفتح /dashboard/ideation من القائمة الجانبية.
  2. يكتب اهتماماته التقنية (10–500 حرف) في حقل multiline.
  3. يضغط «Generate» — تظهر رسائل انتظار متدرجة (5s، 30s) أثناء انتظار Gemini.
  4. تُعرض 3 بطاقات (IdeaSuggestionCard): اسم، هدف (قابل للتوسيع)، تقنيات.
  5. يمكن: حفظ bookmark · إعادة توليد (Regenerate) · اعتماد الفكرة (Adopt & Generate).
  6. أسفل الصفحة: قائمة bookmarks المحفوظة مع إمكانية الحذف.

5.2 API Endpoints

5.3 التحقق في Controller (suggest)

الحقلالقواعد

5.4 خط أنابيب Service — generateSuggestions()

1. Check GEMINI_API_KEY + endpoint 2. sanitizeInput(interests): strip_tags → remove <>{}[] → max 500 chars → collapse whitespace 3. assertMinimumInterestsLength (≥ 10 after sanitize) 4. normalizeExcludeNames (trim, unique) 5. buildPrompt from config/ai.php: detect Arabic → language_instruction inject exclude_section if regenerate 6. callGeminiWithRetry → raw JSON text 7. parseAndValidateResponse → exactly 3 suggestions 8. logRequest → ideation_requests (success/error/timeout) 9. return suggestions[] to Controller

5.5 مخطط JSON المطلوب (Ideation)

{
  "suggestions": [
    {
      "name": "string (max 100)",
      "goal": "string (max 500)",
      "technologies": ["string (max 50 each)", "..."]
    }
  ]
}

قواعد التحقق: suggestions مطلوب · array · size:3 بالضبط · technologies: 1–10 عناصر.

5.6 إعادة التوليد (Regenerate) — exclude_names

عند الضغط على «Regenerate» في الواجهة، يجمع Frontend أسماء الاقتراحات السابقة (previousNames + أسماء الدفعة الحالية) ويرسلها في exclude_names. Service يُدرجها في الـ prompt:

  • عربي: «لا تعِد استخدام أو تُقارِب أسماء المشاريع التالية: …»
  • إنجليزي: «Do NOT reuse or closely resemble these project names: …»

الهدف: اقتراحات جديدة مختلفة دون تكرار الأسماء في نفس الجلسة.

5.7 نظام Bookmarks

القاعدةالتفاصيل
الحد الأقصى10 bookmarks نشطة (archived=false) لكل طالب
عند تجاوز الحدtransaction + lockForUpdate → أقدم bookmark يُؤرشف (archived=true)
التخزينname, goal, technologies (JSON) في bookmarked_ideas
الرسالةإذا أُرشف الأقدم: «Your oldest bookmark was archived to make room.»

5.8 تسجيل الطلبات — ideation_requests

6. ميزة 2: توليد المهام (AI Task Breakdown)

6.1 مسار المستخدم

  1. من تبويب Tasks في تفاصيل المشروع — زر «Generate AI Tasks».
  2. يُشترط وصف المشروع ≥ 20 حرفاً (projectDescription من ProjectDetails).
  3. إذا وُجدت مهام AI سابقاً وكلها pending → يظهر تأكيد SweetAlert لإعادة التوليد.
  4. إذا بدأ الطالب/الفريق أي مهمة AI (status ≠ pending) → الزر يختفي ولا regenerate.
  5. بعد النجاح: المهام الجديدة تُدمج في القائمة المحلية؛ المهام اليدوية تبقى.

6.2 API

MethodRouteBodyMiddleware
POST /api/projects/{project}/generate-tasks { "regenerate": false | true } auth · student · throttle:ai-tasks · ProjectAccess

AITaskController يتحقق من وجود المشروع وProjectAccess::canAccess() قبل استدعاء Service.

6.3 قواعد assertGenerationAllowed()

الحالةregenerate=falseregenerate=true
لا مهام AI✓ مسموح✓ مسموح (يُنشئ جديد)
مهام AI موجودة — كلها pending✗ 422 «Set regenerate=true»✓ يحذف القديم ويُنشئ جديد
مهام AI — واحدة على الأقل in_progress/completed✗ 422✗ 422 «Cannot regenerate after tasks started»

6.4 خط أنابيب generateTasks()

1. Validate description ≥ 20 chars 2. assertGenerationAllowed(project, regenerate) 3. buildPrompt: sanitize title (max 200) + description (max 5000) detect Arabic → language_instruction inject task_count from config 4. callGeminiWithRetry 5. parseAndValidateResponse (1..maxTasks tasks) 6. saveTasksTransactionally: if regenerate → DELETE tasks WHERE ai_generated=true CREATE new tasks: ai_generated=true, status=pending, estimated_hours 7. recordGenerationActivity → ProjectActivity + notifyProjectParticipants 8. logRequest → task_generation_logs 9. return { tasks, replaced_count }

6.5 مخطط JSON المطلوب (Tasks)

{
  "tasks": [
    {
      "title": "string (max 100)",
      "description": "string (max 500)",
      "estimated_hours": 8
    }
  ]
}

قواعد: estimated_hours integer من 1 إلى 40 · عدد المهام من 1 إلى AI_TASK_COUNT (افتراضي 10).

6.6 ما يُخزَّن في جدول tasks

الحقلقيمة AI
title, descriptionمن Gemini
estimated_hoursمن Gemini
ai_generatedtrue
statuspending
project_id, university_idمن المشروع (multi-tenant)
deadlinenull — يحدده الطالب لاحقاً

6.7 ما بعد التوليد — Activity & Notifications

  • ProjectActivity: action_key = tasksAiGenerated أو tasksAiRegenerated · meta: count, replaced.
  • Notification: type task.ai_generated لجميع participants (ما عدا الفاعل).
  • Deep link: /dashboard/projects/{id} في بيانات الإشعار.

7. مسار «اعتماد الفكرة + إنشاء مشروع + توليد مهام»

زر «Adopt & Generate» في IdeaSuggestionCard ينفّذ سلسلة من 3 طلبات:

Step 1: POST /api/project/create { title: suggestion.name, description: suggestion.goal } Step 2: POST /api/projects/{newId}/generate-tasks { regenerate: false } Step 3: navigate → /dashboard/projects/{id}?tab=tasks If Step 2 fails → partial success toast → navigate anyway (project exists, no AI tasks)

هذا يربط Ideation بـ Task Generation في تجربة واحدة للطالب الذي اختار فكرة جاهزة.

8.3 كشف اللغة (Arabic Detection)

Regex Unicode ranges للحروف العربية في النص المدخل. إذا وُجدت → التعليمات والمخرجات (name/goal أو title/description) بالعربية الفصحى، مع بقاء أسماء التقنيات بالإنجليزية (React, Laravel, MySQL…).

9. الأمان، الصلاحيات، وحدود الاستخدام

9.1 Sanitization (قبل إرسال النص لـ Gemini)

  • strip_tags() — إزالة HTML.
  • إزالة < > { } [ ] — تقليل prompt injection.
  • حد أقصى للطول: interests 500 · title 200 · description 5000.
  • طي المسافات المتعددة.

9.2 Rate Limiting

LimiterالحدالمفتاحRoute
ai-ideation10 / ساعةuser_id أو IPPOST suggest-projects
ai-tasks10 / دقيقةuser_id أو IPPOST generate-tasks

9.3 الصلاحيات

  • Ideation + bookmarks: student only (role:student middleware).
  • Task generation: student + يجب أن يكون ProjectAccess::canAccess (عضو مقبول أو owner).
  • Bookmarks: DELETE يتحقق user_id — لا حذف bookmark طالب آخر.

9.4 الخصوصية

نص الاهتمامات لا يُحفظ في DB — فقط hash. وصف المشروع موجود أصلاً في جدول projects (ليس log إضافي للـ prompt). مفتاح API على السيرفر فقط — لا يظهر في Frontend.

12. واجهة المستخدم (Frontend) — تفاصيل إضافية

ProjectIdeation.jsx

  • عداد أحرف {length}/500 · تعطيل زر Generate إذا < 10.
  • رسائل انتظار: generating → stillWorking (5s) → takingLonger (30s).
  • bookmarkedNames Set — تعطيل bookmark مكرر لنفس الاسم.
  • تحميل bookmarks عند mount — failure غير blocking.

TasksTab.jsx

  • aiTasks = filter ai_generated === true.
  • canRegenerateAiTasks = every ai task status === pending.
  • showAiGenerateButton = !hasAiTasks || canRegenerateAiTasks.
  • عند regenerate: manual tasks تبقى · AI tasks القديمة تُستبدل في UI.

14. نقاط للمناقشة في الرسالة العلمية

  • لماذا Gemini خارجياً؟ — تكلفة، صيانة، وعدم الحاجة لبنية ML محلية في مشروع إدارة مشاريع.
  • Config-driven prompts — قابلية تعديل السلوك دون deploy كود Service.
  • Strict JSON validation — طبقة أمان بين LLM غير حتمي وقاعدة بيانات structured.
  • Human-in-the-loop — AI يقترح؛ الطالب يعتمد؛ regenerate محدود بحالة pending.
  • Multi-tenant — مهام AI تحمل university_id مثل باقي الكيانات.
  • Observability — logs (hash, timing, status) دون تخزين prompts كاملة.
  • Rate limits — حماية من إساءة الاستخدام وتكلفة API.
  • ثنائية اللغة — كشف تلقائي للعربية في prompt instruction وليس في واجهة منفصلة.
