3. Account Approval

How it works in PMS: Self-registration is allowed for student and supervisor only. While pending, the user sees the waiting screen with limited access (profile and log out only). Supervisors may select multiple universities — each admin approves or rejects only their own university. After every decision, refreshAccountStatusFromMemberships() sets: active if any membership is active · pending if some memberships are still waiting · rejected only when every membership is rejected. Rejected users see a different screen with a re-register button (same account email can submit a new request).

flowchart TD
    S([Account created]) --> D0{How was account created?}
    D0 -->|Self-registration| D1{Registered role?}
    D0 -->|Created by admin| A0[Set account to active immediately]
    A0 --> A0B[Grant full platform access]
    D1 -->|Student| A1[Set user status to pending]
    D1 -->|Supervisor| A2[Set user status to pending]
    A2 --> A2B[Attach pending membership for each selected university]
    A1 --> A3[Notify university admin]
    A2B --> A3
    A3 --> A4[Show pending approval screen]
    A4 --> A5[Limited access: profile and log out only]
    A5 --> A6[Admin reviews pending user]
    A6 --> D2{User role?}
    D2 -->|Student| D3{Admin decision?}
    D3 -->|Approve| A7[Set student status to active]
    A7 --> A8[Notify student]
    A8 --> A9[Unlock full platform access]
    D3 -->|Reject| A10[Set student status to rejected]
    A10 --> A11[Show rejected account screen]
    A11 --> A12[Offer re-register button]
    D2 -->|Supervisor| D4{Admin decision for this university?}
    D4 -->|Approve| A13[Set this university membership to active]
    D4 -->|Reject| A14[Set this university membership to rejected]
    A13 --> A15[Recalculate status from all memberships]
    A14 --> A15
    A15 --> D5{Resulting account status?}
    D5 -->|At least one active membership| A9
    D5 -->|Still pending memberships| A4
    D5 -->|All memberships rejected| A11
    A13 --> A16[Notify supervisor of approval]
    A14 --> A17[Notify supervisor of rejection]
    A12 --> D6{User re-registers?}
    D6 -->|Yes| S
    D6 -->|No| E1([Stay on rejected screen or log out])
    A0B --> E([End])
    A9 --> E
    A16 --> E
    A17 --> E
    E1 --> E

6. Project Management

10. Project Releases and GitHub
