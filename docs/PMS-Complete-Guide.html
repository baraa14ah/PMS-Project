1. ما هو المشروع؟

PMS هو نظام إدارة مشاريع تخرج جامعية متعدد الجامعات (Multi-Tenant). كل جامعة تعمل كـ tenant مستقل: طلاب، مشرفون، ومديرو جامعة يرون بيانات جامعتهم فقط، بينما مدير المنصة (Super Admin) يرى كل شيء.

الجزءالتقنيةالمجلد
Backend (API)Laravel 10 + Sanctumbackend last/
Frontend (UI)React 19 + Vite + MUI 7frontend/

القدرات الرئيسية: إنشاء مشاريع، دعوات مشرف/طلاب، مهام Kanban، تعليقات، إصدارات، إشعارات، ربط GitHub، وميزتان AI: اقتراح أفكار مشاريع وتوليد مهام تلقائياً عبر Google Gemini.

2. الصورة الكبيرة (Architecture)

Frontend (React Pages + Context + apiFetch) | | HTTP + Bearer Token v Backend routes/api.php -> Controllers -> Services -> Models -> MySQL | +-> Google Gemini (AI) +-> GitHub OAuth

تدفق الطلب النموذجي: المستخدم يتفاعل مع صفحة React → apiFetch يرسل طلباً للـ API → Controller يتحقق من الصلاحيات → Service ينفّذ المنطق → Model يقرأ/يكتب في قاعدة البيانات.

3.5 Multi-Tenancy — TenantScope

يُطبَّق تلقائياً على Project و Task. كل استعلام يُفلتر حسب university_id للمستخدم الحالي. Super Admin يتجاوز الفلتر. محاولة الوصول لـ ID من جامعة أخرى تُرجع 404 (وليس 403).

6. الذكاء الاصطناعي — شرح كامل

6.1 ميزتان منفصلتان

الميزةالغرضالمستخدم
Project Ideation3 أفكار مشاريع من اهتمامات الطالبStudent
Task Generation10 مهام برمجية من وصف المشروعStudent (مالك أو عضو مقبول)

6.2 الإعداد Backend

  • config/services.php — مفتاح Gemini، النموذج، endpoint، timeout، retries
  • config/ai.php — قوالب ideation_prompt و task_breakdown_prompt
  • Rate limits: Ideation 10/ساعة، Tasks 10/دقيقة

6.3 تدفق Ideation

POST /api/ai/suggest-projects [student] → AIIdeationController::suggest() → AIIdeationService::generateSuggestions() 1. sanitizeInput (إزالة HTML/tags) 2. buildPrompt (عربي/إنجليزي تلقائياً) 3. callGeminiWithRetry → Google Gemini JSON mode 4. parseAndValidateResponse → بالضبط 3 اقتراحات 5. logRequest → ideation_requests (hash فقط) → JSON { suggestions, generated_at }

Frontend: ProjectIdeation.jsx + IdeaSuggestionCard.jsx

  • Bookmark: POST /ai/bookmarks (حد 10، أقدم يُؤرشف)
  • Adopt & Generate: ينشئ مشروعاً → يولّد مهاماً → يفتح tab=tasks

6.4 تدفق Task Generation

POST /api/projects/{id}/generate-tasks?regenerate=true|false → AITaskController::generate() → ProjectAccess::canAccess() → AITaskService::generateTasks() 1. description ≥ 20 chars 2. assertGenerationAllowed (لا تكرار بدون regenerate) 3. callGeminiWithRetry 4. saveTasksTransactionally → Task (ai_generated=true) 5. ProjectActivity + Notifications 6. TaskGenerationLog

6.5 قواعد Regenerate

  • أول مرة: 10 مهام AI بحالة pending
  • Regenerate: يحذف مهام AI القديمة فقط إذا كلها pending
  • المهام اليدوية لا تُحذف أبداً

7. المصادقة والأدوار

7.1 Sanctum Token

Login  → POST /login  → token
Request → Authorization: Bearer {token}
Logout → POST /logout → حذف Token

AuthContext.jsx: Token في localStorage، عند reload يستدعي GET /profile، apiFetch يرسل headers تلقائياً.

7.2 دورة حياة المستخدم

Register → status=pending → PendingApproval
Admin approve → status=active → Dashboard
Rejected → PendingApproval (مرفوض)
no university_id → AccountBlocked

8. Design Patterns و SOLID

8.1 الأنماط المستخدمة

Patternأينالشرح
Service Layerapp/Services/*Controllers رفيعة، المنطق في Services
RepositoryProjectRepositoryفصل استعلامات Project (جزئي)
Dependency InjectionLaravel auto-wiringحقن Services في Constructors
Global ScopeTenantScopeفلترة tenant تلقائية
Trait CompositionBelongsToUniversity, ApiResponseTraitإعادة استخدام
HelperProjectAccess::canAccess()مركزية صلاحيات المشروع

8.2 SOLID — أمثلة

مبدأمثال في المشروع
Single ResponsibilityNotificationService للإشعارات، AIIdeationService للـ ideation
Open/Closedtrait BelongsToUniversity لإضافة tenant لـ model جديد
Dependency InversionController → Service → Repository/Model (ليس HTTP مباشرة)
ملاحظة: Repository Pattern غير مكتمل — معظم Services تستعلم Eloquent مباشرة. لا Laravel Policies — الصلاحيات عبر Middleware + ProjectAccess.
